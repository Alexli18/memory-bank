# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

Memory Bank is currently in **early alpha**. Security fixes are applied to the latest version only.

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

To report a vulnerability, use [GitHub Private Vulnerability Reporting (PVR)](https://github.com/Alexli18/memory-bank/security/advisories/new):

1. Go to the **Security** tab of this repository
2. Click **Report a vulnerability**
3. Fill in the details and submit

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

| Action | Timeframe |
| ------ | --------- |
| Acknowledgment of report | Within **48 hours** |
| Detailed response with assessment | Within **7 days** |
| Fix release (if confirmed) | Best effort, depends on severity |

## Disclosure Policy

- We follow **coordinated disclosure**: we ask reporters to keep vulnerabilities private until a fix is released.
- Credit will be given to reporters in release notes (unless they prefer to remain anonymous).
- If we are unable to address the issue within 90 days, the reporter may disclose it publicly.
